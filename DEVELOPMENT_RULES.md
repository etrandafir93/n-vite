# Development Rules

- Apply UI/UX best practices on every frontend change (clarity, consistency, responsiveness, accessibility, and compact/scannable layouts where appropriate).
- Any new frontend text field, label, message, or CTA must be integrated into i18n and translated in all supported locales (`en`, `ro`, `uk`, `es`).
- Backend changes must follow SOLID principles and clean code practices (single responsibility, readable structure, meaningful naming, and maintainable abstractions).
- Every change (frontend or backend) must include a version update before commit/push.
